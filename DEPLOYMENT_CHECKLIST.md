# Vercel 배포 체크리스트

배포 전에 다음 항목들을 확인하세요.

## ✅ 프로젝트 준비

- [ ] Next.js 프로젝트가 정상적으로 작동하는지 확인
- [ ] `npm run build` 명령어로 빌드가 성공하는지 확인
- [ ] `npm run dev`로 로컬 개발 서버가 정상 작동하는지 확인
- [ ] 모든 페이지와 컴포넌트가 정상적으로 렌더링되는지 확인
- [ ] 이미지, 폰트 등 정적 자산이 올바르게 로드되는지 확인

## ✅ 코드 품질

- [ ] 콘솔 에러가 없는지 확인
- [ ] TypeScript 타입 오류가 없는지 확인 (TypeScript 사용 시)
- [ ] ESLint 경고를 확인하고 수정
- [ ] 불필요한 주석이나 디버그 코드 제거

## ✅ 환경 변수

- [ ] 필요한 환경 변수를 `.env.local`에 설정
- [ ] Vercel 대시보드에 환경 변수 추가 준비
- [ ] 민감한 정보(API 키 등)가 코드에 하드코딩되지 않았는지 확인

## ✅ Git 저장소

- [ ] `.gitignore` 파일이 올바르게 설정되어 있는지 확인
- [ ] `node_modules`, `.next` 등 불필요한 파일이 커밋되지 않았는지 확인
- [ ] 초기 커밋이 완료되었는지 확인
- [ ] 원격 저장소(GitHub 등)에 푸시 완료

## ✅ Vercel 설정

- [ ] Vercel 계정 생성 및 로그인 완료
- [ ] GitHub/GitLab 연동 완료
- [ ] 프로젝트 선택 및 배포 설정 확인

## ✅ 배포 후 확인

- [ ] 배포된 사이트가 정상적으로 로드되는지 확인
- [ ] 모든 페이지가 정상적으로 작동하는지 확인
- [ ] 모바일 반응형이 올바르게 작동하는지 확인
- [ ] 이미지와 정적 자산이 올바르게 로드되는지 확인
- [ ] 커스텀 도메인 연결 (선택사항)

## ✅ 성능 최적화

- [ ] 이미지 최적화 (Next.js Image 컴포넌트 사용)
- [ ] 불필요한 의존성 제거
- [ ] 번들 크기 확인 및 최적화

---

## 🚀 배포 명령어 요약

```bash
# 1. 로컬 빌드 테스트
npm run build

# 2. Git 커밋 및 푸시
git add .
git commit -m "배포 준비 완료"
git push origin main

# 3. Vercel에서 자동 배포 확인
# (Vercel 대시보드에서 배포 상태 확인)
```

---

## 📝 참고사항

- Vercel은 무료 플랜에서도 충분히 포트폴리오 사이트를 호스팅할 수 있습니다.
- 커스텀 도메인 연결도 무료 플랜에서 가능합니다.
- 자동 배포는 `main` 브랜치에 푸시할 때마다 트리거됩니다.
